# M9.1 赛题对齐补充实验报告

`supplementary_competition_alignment_ablation`，`conclusion_scope` 为
`fixed_task_fixed_model_supplementary_analysis`。本报告只使用通过 Strict
Verifier 的 `public/tasks` final records 聚合结果，不读取 private tests、
candidate code、raw response、private traceback 或模型服务日志。

## 通信与模型开销

S1 平均产生 1 条文本 Agent 消息和 363.45 个 Agent 文本字符。S2--S4 平均产生
2.2 条 Protocol 消息；S2、S3、S4 的平均 Protocol payload bytes 分别为
620.8、627.8、638.6，header bytes 均为 87.0，reference_id_count 均为 3.0。

`agent_text_tokens`、`protocol_payload_tokens`、`repeated_context_bytes`、
`deduplicated_context_bytes`、`capability_handshake_count` 与
`capability_handshake_bytes` 为 `unavailable`。握手在 Runner 差分计量基线前
发生，正式记录中的零值不能解释为未握手。因此，S2 相比 S1 是否降低同单位的
Agent 通信 Token 或字符，不能由当前公开记录证明；也不能将 Protocol payload
bytes 与 Text 字符直接作为等价单位比较。

与 S1 相比，S2 的总 Token 和 Provider 延迟均增加。M9.1 不支持“Protocol V2
一定节省通信或模型开销”的结论。

## StateVector V2

S3 与 S4 均记录平均 1.0 个 StateVector、33.0 个 StateVector bytes、9.0 个
state reference bytes，以及 130.0 个 equivalent text state bytes。由此得到
state compression ratio 为 0.7462。平均编码耗时约 4.1 微秒，解码耗时约
20.5 微秒，`invalid_state_count` 为 0。

该结果表明 V2 状态载荷小于记录的等价文本状态载荷；它只说明状态传递压缩，
不说明 StateVector 必然提高任务质量。

## GatedSharedMemory V2

S4 的每任务平均 Memory 指标为：query 1.00、candidate 0.95、hit 0.60、
accept 0.95、reject 0、abstain 0.40、reuse 0.95、injected 0.95、injected
tokens 0.95、injected bytes 17.1、write 1.00、success write 0.55、eviction 0。

`memory_effective_reuse_rate` 为 `unavailable`，不能以实际引用后的可评测
结果推断因果性能提升。S4 相比 S3 的任务成功率差为 -0.05，显示在本次固定
条件下仍存在负迁移风险；该结果不触发换题、删记录或基于 hidden tests 调参。

## 结论与限制

1. M9.1 的有效输入为 240/240 严格验证的公开 final records，且未发现公开泄漏。
2. S3 在本次固定任务与固定模型条件下取得最高任务成功率 0.60；这不是对其他
   模型、任务或温度的泛化结论。
3. StateVector V2 记录到 33 bytes 对 130 bytes 等价文本状态的压缩，但质量变化
   仍需与 Protocol 和模型输出共同解释。
4. S4 产生真实 Memory query、hit、accept、abstain 与 reuse 记录，但相对 S3
   出现质量负差异，不能声称 Memory V2 已消除负迁移。
5. 由于部分通信层字段为 `unavailable`，本报告不声称 CompactProtocol V2 已被
   证明降低 Agent 通信 Token 或字符。
# 冻结行为与当前主分支说明

本报告描述 freeze `c79fd4826627bf61faf5d90540a014d243a59edd` 的历史 M9.1 行为与正式聚合，不因当前主分支工程修复而改写。当前主分支已在 Executor 边界补上 StateVector 与已接受安全 Memory 引用的解析和单次语义注入；该修复没有重新生成 M9.1 正式记录，也不构成冻结实验已验证的功能。

报告中的 `memory_injected_tokens` 为 whitespace-based token estimate，不是 Llama tokenizer 或 Provider `prompt_tokens`，不应作同口径比较。StateVector V2 的固定编码长度为 33 bytes；约 130 bytes 等价文本和约 0.7462 压缩率仅表示状态载荷压缩，不表示质量提升或 Prompt Token 同比例下降。
