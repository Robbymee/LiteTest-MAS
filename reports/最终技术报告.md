# 最终技术报告

## 范围

本报告汇总已冻结 M9 与独立 M9.1 的公开聚合结果。M9 freeze 为 `cc7aac0417afb6acab47baaf7449459692fa9444`；M9.1 freeze 为 `c79fd4826627bf61faf5d90540a014d243a59edd`，其 Spec SHA 为 `3ad520c75bb66e8a4617daa64d6824183cbeaa5a1e1cb01dcd50035f145231f6`。两项实验均为 240/240 final records，并通过公开 Strict Verifier。

## M9 结论

M9 中 G2 Protocol 的任务成功率为 0.75，高于 G1 Text 的 0.60；G2-G1 配对差为 +0.15，95% CI [+0.0333, +0.2671]。但 G2 的平均总 Token 为 371.45、高于 G1 的 211.90，平均 Provider 延迟为 2.1852 秒、高于 G1 的 0.9992 秒。G3 相对 G2 没有成功率增益。G4 发生真实 Memory 读取、命中和复用，但相对 G3 有 -0.05 的任务成功差异，负结果保留。

## M9.1 结论

M9.1 的 S1/S2/S3/S4 任务成功率依次为 0.55、0.55、0.60、0.55。S3 的 StateVector V2 压缩率为 0.7462，说明记录的状态载荷小于等价文本状态；这不等于质量必然提升。S4 记录了 query、hit、accept、abstain 和 reuse，但相对 S3 的质量下降不能被解释为 Memory V2 消除负迁移。部分通信 Token 与重复上下文字段为 `unavailable`，因此不声称 Protocol V2 已被证明节省通信。
