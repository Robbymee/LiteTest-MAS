# 最终技术报告

## 项目与范围

LiteTest-MAS 是面向代码任务生成、测试执行与私有评测的轻量多智能体协作实验平台。Planner、Retriever、Executor、Summarizer 等为逻辑角色，不等同于多个独立 LLM 互相交流。仓库同时提供确定性 pytest 测试生成 Demo 和本地 LLM 代码生成消融实验；M9/M9.1 每个任务主要执行一次候选代码生成请求。

所有正式结论只适用于预注册的固定任务、模型、参数和评测边界。private tests、候选代码和原始响应不进入公开报告。

## 核心结果

| 实验 | 冻结版本 | 结果 | 结论 |
| --- | --- | --- | --- |
| M9 | `cc7aac0417afb6acab47baaf7449459692fa9444` | G1/G2/G3/G4 成功率为 0.60/0.75/0.75/0.70。 | G2 相对 G1 提高任务成功率，但总 Token 与 Provider 延迟增加；G3 相对 G2 无额外成功率收益；G4 有真实 Memory 读、写、命中与复用，但相对 G3 为负差异。 |
| M9.1 | `c79fd4826627bf61faf5d90540a014d243a59edd` | S1/S2/S3/S4 成功率为 0.55/0.55/0.60/0.55。 | S3 在固定条件下最好；S4 有真实 query、hit、accept、abstain 和 reuse，但相对 S3 下降。 |

M9.1 的 StateVector V2 固定为 33 bytes，对应约 130 bytes 等价文本表示，压缩率约 0.7462。这是 inter-agent 状态载荷压缩证据，不是 StateVector 提高质量的因果证明，也不代表 LLM Prompt Token 同比例下降。

Protocol V2 尚未证明降低同口径 Agent 通信 Token 或字符。Memory 注入 Token 使用 `whitespace_v1` 估算，不能与 Provider 返回的 `prompt_tokens` 直接比较。`unavailable` 保持其原始含义，不能填零或据此反推指标。

## 当前主分支与冻结实验

M9 与 M9.1 结论分别绑定上述 freeze SHA。当前主分支可包含冻结实验之后的工程修复，例如执行端对 StateVector 和 Memory 引用的解析与实际 Prompt 注入。此类修复不反向改变冻结实验记录、聚合、Spec、Strict Verifier 或历史结论，也不表示其已在冻结实验中验证。
