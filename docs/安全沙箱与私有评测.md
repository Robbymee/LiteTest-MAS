# 安全沙箱与私有评测

## 数据边界

Agent 只能接收公开任务描述、签名和入口信息。`hidden_reference_tests`、canonical solution、candidate code、raw response、private traceback、凭据和模型服务内部路径不得进入 Prompt、Protocol、StateVector、SharedMemory、公开记录或 Dashboard。

## 执行方式

Candidate Parser 在公开 Runner 链中解析候选。Private Sandbox 以受限子进程执行评测，使用 `shell=False`、临时目录、超时和有界输出；公开结果只保留解析状态、Sandbox 完成状态、official-test 计数、失败类别和泄漏审计状态。

## 审计

Strict Verifier 校验公开计划、记录身份、final 数量、重复、缺失和 completion marker。`scripts/audit_competition_delivery.py` 只审计文档与交付物，不读取任何私有评测目录。安全边界是数据隔离约束，不将受限子进程表述为容器级安全边界。
