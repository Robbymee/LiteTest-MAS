# 关联连续任务设计说明

## 任务来源与顺序

M9 与 M9.1 均从冻结公开 manifest 读取 MBPP 和 HumanEval+ 的真实任务身份、两个任务组及组内顺序。每个数据集有两个连续组，每组五项任务；每个实验组、数据集和 seed 按该固定顺序完成十项任务，不由后处理重命名或重排。

## 可复用结构

关联组内任务共享函数测试、边界输入、候选解析和 Sandbox 执行等工作流。可复用的是公开的过程摘要和脱敏记忆引用，不能复用 private tests、candidate code、canonical solution 或 raw response。

## Memory 边界

Memory 实例按 `dataset + group_id + seed + experiment_id` 隔离，在组边界 reset。M9 V1 的 hit/reuse 仅表示读取或引用事件；M9.1 V2 区分 query、candidate、hit、accept、reject、abstain 与 reuse。有效复用仅表示引用后有可评测结果，不代表因果提升。

## 结果解释

组内轮次、真实任务 ID 和复用计数以冻结 Spec 和公开聚合为准。未被公开记录的门控决定、失败原因或跨组复用不得从任务名称、Token 或成功率推断。
