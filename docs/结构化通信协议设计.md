# Compact Protocol V2 设计

## 范围

`compact_protocol_v2` 是独立模块，不替换 M9 冻结使用的 `protocol_v1`、`ProtocolAdapter` 或 Text Mode。当前阶段只使用合成 fixtures 和 Mock 范围测试，不接入正式 M9 或 M9.1 运行。

## 握手与能力

每个 sequence 只能发送一次握手。握手协商 `compact_protocol_v2`，并一次性发布 `CapabilityRegistry` 中的能力描述和动作集合。接收端建立 remote capability ID 到本地 capability ID 的映射，可按动作执行 capability discovery；未知版本被明确拒绝。

## 紧凑编码

消息采用稳定 UTF-8 JSON。`v` 和 `t` 保留版本与类型，动作消息使用 `a`、`s`、`r`、`c`、`q` 表示 action、sender、receiver、capability ID、task reference。空输入、空结果和默认值不写入载荷。

静态任务通过一次 `task_register` 注册，后续动作只传 `task_ref`。重复参数通过 `reference_register` 注册，后续只传 `reference_id`。StateVector 与 Memory 只传 `state_vector_id` 和 `memory_id`，不在协议层重复传输其内容。

## 安全与审计

注册表仅在本地保存恢复所需的公开值。审计输出只包含能力描述、引用内容摘要哈希和字节计数，不输出 task 原文、candidate code、private tests、raw responses、凭据或 private traceback。协议编码会拒绝相关私有字段。

## 指标

`ProtocolMetrics` 记录握手次数与字节、普通 payload 次数与字节、header bytes 和 reference ID 数量。它们是 V2 实现的通信层指标，不可反推为 M9 已冻结实验的历史指标。
