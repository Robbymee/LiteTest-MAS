# 实验结果与统计分析

## M9 正式结果

M9 的 G1/G2/G3/G4 任务成功率分别为 0.60、0.75、0.75、0.70。G2-G1 的配对任务成功率差为 +0.15，95% CI 为 [+0.0333, +0.2671]；G3-G2 为 0.00；G4-G3 为 -0.05。Protocol V1 与更高成功率同时伴随更高总 Token 和 Provider 延迟，不能据此声称通信更省。StateVector V1 未提高成功率；SharedMemory V1 有真实复用但存在负迁移风险。

## M9.1 补充结果

M9.1 的 S1/S2/S3/S4 任务成功率为 0.55、0.55、0.60、0.55。S3 在本次固定条件下最高，S4 相对 S3 的 -0.05 负差异保留。StateVector V2 记录 33 bytes 对 130 bytes 等价文本状态，压缩率为 0.7462；部分通信层字段仍为 `unavailable`，不能证明 CompactProtocol V2 已降低同单位通信开销。

## 统计口径

Bootstrap 使用 `bootstrap_seed=20260711`、2000 次重采样和 95% CI。M9 另有按 `dataset + task_id` 聚类的敏感性分析。`unavailable` 表示缺少可恢复来源，不是零值，也不能通过总 Token 或质量结果补推。
