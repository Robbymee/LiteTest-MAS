# 匿名化与赛事提交检查

## 禁止公开的内容

提交前检查学校、教师、学生、单位、Windows 用户目录、模型绝对路径、API key、private tests、candidate code、raw response、canonical solution 与 private traceback。公开 Dashboard、报告和文档只引用脱敏聚合数据。

## 检查命令

```bash
python scripts/audit_competition_delivery.py --root . --allow-incomplete
git diff --check
git status --short
```

审计器检查 Markdown 编码、中文正文、链接、常见凭据形态、个人绝对路径、必需中文文档和受保护版本。它不替代人工检查，也不读取私有评测内容。

## 发布约束

保留 `v1.0.0-experiment`，不得覆盖既有 M9/M9.1 结果或冻结提交。只有严格交付审计通过、真实赛事材料齐备且双平台验收完成后，才评估创建 `v1.1.0-competition`。
