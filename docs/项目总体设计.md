# 项目总体设计

## 定位

LiteTest-MAS 是面向代码任务生成、测试执行与私有评测的轻量多智能体协作实验平台。它研究公开任务上下文在 Text、结构化 Protocol、StateVector 和 SharedMemory 之间的组织方式，以及这些方式与质量、通信成本和状态复用的关系。

## 架构边界

`agents/` 中的 Planner、Memory/Retriever、TestGen、Executor、Summarizer 是逻辑角色。角色编排不等同于多个独立 LLM 相互对话；M9/M9.1 的每个任务主要执行一次候选代码生成请求。

公开任务信息可进入协议上下文。Candidate Parser 只解析模型候选，Private Sandbox 在隔离边界使用 official tests，公开记录只保存脱敏结果和聚合指标。hidden tests、candidate code、raw response、canonical solution、private traceback 和凭据不进入公开上下文。

## 两条运行链

### 确定性 pytest 测试生成 Demo

PlannerAgent → MemoryAgent → TestGenAgent → ExecutorAgent → SummarizerAgent。TestGenAgent 基于公开任务中的确定性 cases 生成 pytest，用于演示协作编排与测试执行。

### M9/M9.1 代码生成实验

公开 MBPP/HumanEval+ 任务 → Text 或 Protocol 上下文 → 本地 LLM 候选 Python 实现 → Candidate Parser → Private Sandbox → official-test 聚合。M9 使用 V1 组件，M9.1 使用独立 V2 组件；两者冻结结果、Spec 与报告不得混写。

## V2 执行边界

CompactProtocol V2 在 Agent 间传输 task、StateVector 和 Memory 的引用 ID。当前主分支的 Executor 通过注册表解析 StateVector bytes，并仅向模型注入一次紧凑公开状态；它只解析当前隔离 scope 内、已通过门控的安全 Memory 摘要。此工程修复发生在 M9.1 冻结提交之后，不能倒写为冻结实验已验证的行为。
