# Gated SharedMemory V2 设计

## 门控与元数据

`gated_shared_memory_v2` 的记忆单元包含 `memory_id`、`source_agent`、`created_at`、`task_topic`、`summary`、`tags`、`task_group`、`task_type`、`provenance`、`confidence`、`success_status`、`reuse_count`、`last_used_at` 和 `schema_version`，并保存 dataset、seed 和 experiment scope。

默认只接受 `success_status=success` 的记忆进入注入候选；失败经验可以保留并标记，但不默认注入。检索使用 tag overlap、关键词 overlap 和 task type match 的轻量评分，固定 `top_k`、`relevance_threshold`、`confidence_threshold` 和 `token_budget`。无满足条件的候选时主动 abstain，不为凑 hit 强制注入。

## 隔离与复用

Memory 实例按 dataset、task_group、seed 和 experiment_id 隔离；同一任务不能读取自己刚写入的记忆。`reuse` 只在下游实际引用 memory ID 时增加复用计数，`effective reuse` 只表示引用后任务产生了最终可评测结果，不解释为因果性能提升。

## 安全边界

写入和检索拒绝 hidden tests、candidate code、raw response、canonical solution、expected output、private traceback、凭据等字段。当前实现只使用合成 fixtures，不修改 V1 SharedMemory，不接入正式 Runner 或 M9.1 实验。
