# StateVector V2 设计

## 目标

`state_vector_v2` 使用标准库 `struct` 将完成任务所需的最小状态编码为固定 32 字节 bytes，替代 Agent 间重复的完整状态文本。它是独立实现，不修改 M9 冻结的 StateVector V1。

## 字段与流程

向量包含 `schema_version`、`phase`、`source_role`、`target_role`、`progress_code`、`artifact_flags`、`error_code`、`retry_count`、`confidence`、`state_reference` 和 `memory_reference_count`。上游生成向量并编码，下游先校验 magic、长度、版本和枚举，再解码并使用字段；状态引用只传短引用，不嵌入完整文本。

编码使用网络字节序和固定格式，Windows 与 Linux 使用相同 bytes。`equivalent_text_state_bytes` 表示同一最小字段的公开文本表示，`compression_ratio = 1 - state_vector_bytes / equivalent_text_state_bytes`，仅在等价文本字节数大于零时计算。

## 边界

下游不得同时接收完整文本状态和同义 StateVector。V2 当前只用合成 fixtures 验证编码、解码、错误状态、跨平台稳定性和 V1 关闭时行为，不接入正式 Runner，不调用真实 LLM，也不读取 private tests 或 candidate code。
