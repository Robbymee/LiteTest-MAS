# StateVector V2 设计

## 编码

`state/vector_v2.py` 使用标准库 `struct` 和格式 `>2sBBBBHBBBf16sH` 编码最小公开状态。该布局固定为 **33 bytes**，使用网络字节序，并校验 magic、schema、枚举和字段范围。

字段包括 `schema_version`、`phase`、`source_role`、`target_role`、`progress_code`、`artifact_flags`、`error_code`、`retry_count`、`confidence`、`state_reference` 和 `memory_reference_count`。它不修改 M9 冻结的 StateVector V1。

## 传输与接收

协议层仅传 `state_vector_id`，不重复传输完整自然语言状态。当前主分支的 Executor 通过注册表查找 bytes、严格 decode 并验证 `generation` phase 与 `executor` target role，再构造一次紧凑公开状态块。原始二进制 bytes 不要求 LLM 理解，也不与同义完整文本并列注入。

## 时间与证据范围

V2 最初由独立单元测试验证；后续已接入 M9.1 冻结实验。M9.1 记录了 33 bytes StateVector 与约 130 bytes 等价文本表示，压缩率约为 0.7462。这证明的是 inter-agent 状态载荷压缩。

该结果不表示 StateVector 必然提高任务质量，也不表示 LLM Prompt Token 会同比例下降。当前主分支的执行端引用解析是冻结实验后的工程修复，历史 M9.1 报告仍只描述 `c79fd4826627bf61faf5d90540a014d243a59edd` 的冻结行为。
