# 多智能体角色与协作流程

## 真实调用链

当前 Runtime 的公开调用顺序是 PlannerAgent、MemoryAgent、TestGenAgent、ExecutorAgent、SummarizerAgent。系统通过 `agent_collaboration.json` 记录 `role_id`、`capability_id`、`action`、`input_reference`、`output_reference`、`started_at`、`completed_at` 和 `status`。

| 逻辑角色 | 当前实现 | 公开动作 | 证据边界 |
| --- | --- | --- | --- |
| Planner Agent | `agents/planner.py` | `planner.plan` | 产生任务计划和输入目标 |
| Retriever Agent | `agents/memory_agent.py` | `memory.store_context` | 当前是确定性上下文存储，不是独立 LLM 检索调用；V2 门控检索尚未接入 Runtime |
| TestGen Agent | `agents/testgen.py` | `executor.generate_candidate` | 生成公开测试文件；属于逻辑流程角色 |
| Executor Agent | `agents/executor.py` | `executor.sandbox` | 调用受限 Sandbox 执行 pytest |
| Summarizer Agent | `agents/summarizer.py` | `summarizer.public_summary` | 汇总公开结果并写入 summary |

## 角色与 LLM 调用

逻辑 Agent 角色表示职责边界和协作事件，不等于一次独立 LLM 请求。当前这条 smoke Runtime 中这些角色主要由确定性 Python 代码实现；LLM 请求次数由具体 Runner 和 Backend 记录，不能根据角色事件数量推断。P8 不把一个函数调用虚构为独立智能体。

## Dashboard 边界

M10 Dashboard 是已冻结发布产物，本阶段不覆盖或改写它。P8 新增的角色证据先以 `agent_collaboration.json` 和本中文档保存；后续新的 Dashboard 版本必须在独立交付阶段生成并重新审计，不能改变 M10 历史结果。
